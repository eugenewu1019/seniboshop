## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please help us by reporting it responsibly.

### How to Report

**Please do NOT create a public GitHub issue for security vulnerabilities.**

Instead:

1. **Email**: Send details to the repository owner via GitHub
2. **Include**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- **Acknowledgment**: We'll respond within 48 hours
- **Updates**: You'll receive regular updates on our progress
- **Fix Timeline**: We aim to release patches within 7 days for critical issues
- **Credit**: You'll be credited in the release notes (unless you prefer to remain anonymous)

### Security Best Practices

When using this project:

- Keep your browser updated
- Use HTTPS when deploying
- Sanitize user inputs if extending the code
- Review third-party dependencies (Tailwind CDN, Google Fonts, etc.)

## Disclosure Policy

We believe in responsible disclosure:

1. Report the vulnerability privately
2. Give us reasonable time to fix it
3. Don't exploit the vulnerability
4. We'll publicly disclose after a fix is released
